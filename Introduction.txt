Types of SQL Languages & Commands :


	DDL

	DML

	DCL

	TCL

	DQL




1. DDL : Data Definition Language 

	Defines the structure of a Database

	- CREATE :

		Create table,database etc

	- RENAME :

		Rename table or columns

	- ALTER :

		Modifies the table Structure

	- DROP  :

		Delete a column ,table/database permanently

	- TRUNCATE :

		Remove all records from table but keeps the structure.


2. DML : Data Manipulation Language

	Works with the data inside the table

	- INSERT :

		Add new Records

	- UPDATE :

		Modifies Existing Records

	- DELETE :

		Removes Specific Records


3. DCL : Data Control Language


	- GRANT :

		Give users access to the database.

	- REVOKE :

		Removes access from users


4. TCL : Transaction Control Language :


	- COMMIT :

		save changes permanently

	- ROLLBACK :

		cancel uncommited changes

	- SAVEPOINT :

		creates a temporary checkpoint in a transactions


5. DQL : Data Query Language 


	- SELECT :

		Retrieves data from tables.

Data Types :

	1. Numeric Data
	2. Text Data
	3. Date & Time
	4. Binary Data
	5. Boolean Data
	6. Special Data


Numeric Data Type :

	INT          --> Whole Numbers [1,2,30,100,etc...]
	SMALLINT     --> Small Numbers
	BIGINT       --> Large Numbers
	DECIMAL(p,s) --> Decimal Numbers(23.456,67.987)


Character and (Text\String) Data Type : "werwe", "456"


	CHAR(n)      --> Fixed Length Text
	VARCHAR(n)   --> Variable - Length Text
	TEXT	     --> Stores a Long Text


Date & Time Data Type :

	DATE         --> Stores Only Date(YYYY-MM-DD)
	TIME         --> Stores Only Time(HH-MM-SS)
	DATETIME     --> Stores both date and time
	TIMESTAMP    --> Date and time with timezone support



Binary Data Type :

	BINARY    --> Fixed length binary
	VARBINARY --> Variable-length binary

Boolean Data Type :

	Boolean Stores TRUE or FALSE

Special Data Types :

	UUID --> Universally Unique Identifier
	JSON --> Stores JSON Formatted data
	XML  --> Stores XML Formatted data



Constraints :

	- Rules for Accurate Data Storage



	1. Primary Key :

		Ensure Unique & not-null values in coulmn

	2. Unique :

		No duplicate values allowed

	3. Foregin Key:

		Links a Column to another table

	4. Check :

		Ensure a column follows a certain condition

	5. Not Null :

		Prevent  Empty Values

	6. Default :

		set default value if no value is given

	7. Auto Increment :

		Automatically generate unique numbers
